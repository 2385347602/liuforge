## 1. 使用 `as const` 获得精确类型

声明常量对象时，TypeScript 默认会推导出相对宽泛的类型。使用 `as const` 可以让属性变成只读的字面量类型。

```typescript
const config = {
  mode: 'production',
  port: 3000,
} as const;

// config.mode 的类型是 'production'
```

## 2. 用 `satisfies` 验证类型

`satisfies` 可以验证一个表达式是否满足某个类型，同时保留表达式本身更精确的推导结果。

```typescript
type RGB = readonly [red: number, green: number, blue: number];

const blue = [0, 0, 255] satisfies RGB;
// blue 的类型仍然保留为 readonly [0, 0, 255]
```

## 3. 模板字面量类型

模板字面量类型可以描述更精确的字符串约束，适合事件名、CSS 单位、路由参数等场景。

```typescript
type EventName = `on${Capitalize<string>}`;
type CSSUnit = `${number}${'px' | 'rem' | 'em'}`;
```

## 4. 组合内置工具类型

`Pick`、`Partial`、`Readonly`、`Record` 等工具类型可以组合出更清晰的业务约束。

```typescript
interface User {
  id: string;
  name: string;
  email: string;
  avatar: string;
}

type UpdateUserPayload = Partial<Pick<User, 'name' | 'avatar'>>;
```

## 5. 用 `never` 做穷尽检查

在 `switch` 或条件分支中，`never` 可以帮助你发现没有处理到的新状态。

```typescript
type Status = 'idle' | 'loading' | 'success' | 'error';

function handleStatus(status: Status) {
  switch (status) {
    case 'idle':
      return '等待中...';
    case 'loading':
      return '加载中...';
    case 'success':
      return '加载成功';
    case 'error':
      return '加载失败';
    default: {
      const exhaustive: never = status;
      return exhaustive;
    }
  }
}
```

> 类型系统不是束缚，而是写给未来自己的文档和测试。
